# Workfront Admin Consoleのランディングページ

Adobe Admin Consoleへの組織のオンボーディングの準備 [&#x200B; 移行チェックリスト &#x200B;](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/admin-in-admin-console/prep-for-admin-console.html?lang=ja):

**移行チェックリスト**

* Workfrontを追加するAdobe Admin Consoleを特定します。

  既に他のAdobe製品がある場合は、現在のAdobe Admin ConsoleにAdobe Workfrontを追加する意思があることを確認してください。 詳細については、[&#x200B; このリンク &#x200B;](https://helpx.adobe.com/jp/enterprise/using/admin-console.html) を参照して、Adobe製品を管理するためのベストプラクティスを理解してください。

* デプロイメントのヘルプについては、[&#x200B; 次のリンク &#x200B;](https://helpx.adobe.com/jp/enterprise/using/deployment-planning.html) を参照してください。
* シングルサインオンを利用する場合は、新しいAdobe Admin Consoleを既存の SSO プロバイダーに接続する必要があります。
* 重複するメールを削除し、有効なメールアドレスのみを使用するようにします。
* カスタム API 統合を使用している場合は、ユーザーがアクセスできる有効なメールアドレスを持っていることを確認します。

  Workfrontでは、すべてのカスタム統合を更新して [OAuth2 を利用 &#x200B;](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/configure-integrations/create-oauth-application.html?lang=ja) することをお勧めします。

コンソールを設定したら、このリンクを参照して、Adobe Admin Console内で [&#x200B; ユーザーの管理方法 &#x200B;](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/add-users/create-manage-users/admin-console.html?lang=ja) を確認してください。
