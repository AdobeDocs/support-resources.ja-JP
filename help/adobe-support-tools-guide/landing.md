# Workfront Admin Consoleのランディングページ

Adobe Admin Consoleへの組織のオンボーディングの準備 [ 移行チェックリスト ](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/admin-in-admin-console/prep-for-admin-console.html):

**移行チェックリスト**

* Workfrontを追加するAdobe Admin Consoleを特定します。

  既に他のAdobe製品がある場合は、現在のAdobe Admin ConsoleにAdobe Workfrontを追加する意思があることを確認してください。 詳細については、[ このリンク ](https://helpx.adobe.com/jp/enterprise/using/admin-console.html) を参照して、Adobe製品を管理するためのベストプラクティスを理解してください。

* デプロイメントのヘルプについては、[ 次のリンク ](https://helpx.adobe.com/enterprise/using/deployment-planning.html) を参照してください。
* シングルサインオンを利用する場合は、新しいAdobe Admin Consoleを既存の SSO プロバイダーに接続する必要があります。
* 重複するメールを削除し、有効なメールアドレスのみを使用するようにします。
* カスタム API 統合を使用している場合は、ユーザーがアクセスできる有効なメールアドレスを持っていることを確認します。

  Workfrontでは、すべてのカスタム統合を更新して [OAuth2 を利用 ](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/configure-integrations/create-oauth-application.html) することをお勧めします。

コンソールを設定したら、このリンクを参照して、Adobe Admin Console内で [ ユーザーの管理方法 ](https://experienceleague.adobe.com/docs/workfront/using/administration-and-setup/add-users/create-manage-users/admin-console.html) を確認してください。
