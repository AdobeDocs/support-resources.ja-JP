# Global Admin Consoleでの組織ポリシーの更新

**適用先：** Enterprise

Global Admin Consoleで、グローバル管理者が組織とその子のポリシーを設定および変更する方法について説明します。

>[!NOTE]
>
>[Global Admin Console](https://helpx.adobe.com/jp/enterprise/global-admin-console/adopt-global-administration.html)で、階層から組織を選択し、「**ポリシー**」タブに移動して、ポリシーを許可または禁止またはロックします。
>
> [Global Admin Consoleにログイン &#x200B;](https://global-admin-console.adobe.com/)

ポリシーは組織に関連付けられ、その組織で実行できる操作を制限します。 ポリシー値を設定すると、その時点からアクションが制限または有効になります。
例えば、**要求ドメイン** ポリシーが&#x200B;*許可されていません*&#x200B;に設定されている場合、追加のドメインは要求できませんが、ポリシー値を設定する前に要求されたドメインは影響を受けません。

## ポリシーの設定

組織のポリシーを変更するには、次の操作を行います。

1. Global Admin Consoleで、[編集する組織](https://helpx.adobe.com/jp/enterprise/global-admin-console/overview.html)を選択し、**[!UICONTROL ポリシー]** タブに移動します。
1. 関連するポリシーのトグルを選択して、ポリシーを許可または禁止します。 また、ポリシーをロックして、[選択した組織](https://helpx.adobe.com/jp/enterprise/global-admin-console/overview.html)またはその親組織以外のユーザーが変更またはロック解除できないようにすることもできます。
1. ポリシーをロックするには、**[!UICONTROL ロック]** ![&#x200B; ロック &#x200B;](./assets/lock.png) アイコンを選択します。 ロックにカーソルを合わせると、選択した組織の名前が表示されます。 [&#x200B; ポリシーロック &#x200B;](#policy-locks)の詳細をご覧ください。
1. 組織の編集が完了したら、「**[!UICONTROL 保留中の変更を確認]**」を選択します。 レビュー後、**[!UICONTROL 変更を送信]**&#x200B;から[実行](https://helpx.adobe.com/jp/enterprise/global-admin-console/execute-jobs.html)を選択します。

## ポリシーロック {#policy-locks}

ポリシーがロックされている場合、その値はポリシーがロック解除されるまで変更できません。 Global Admin Consoleは、組織ピッカーの[選択した組織](https://helpx.adobe.com/jp/enterprise/global-admin-console/overview.html)を、ポリシーがロックされた組織として記憶します。 選択した組織またはツリー内の上位の組織のグローバル管理者には、ポリシーのロックを解除する権限があります。 スコープがその組織よりも低いグローバル管理者には、ポリシー値のロックを解除して変更する権限がありません。

ロックダウン環境を作成するには、子組織に必要なポリシー値を設定してからロックします。 これらの子組織のグローバル管理者は、ポリシー値を編集できません。

### 例：ロックダウン環境

*Acme Division*&#x200B;のグローバル管理者であるElissaが子組織&#x200B;*Marketing*&#x200B;と&#x200B;*Engineering*&#x200B;を作成し、Robertを&#x200B;*Marketing*&#x200B;のグローバル管理者として、Sarahを&#x200B;*Engineering*&#x200B;のグローバル管理者として追加した場合。 次に、彼女は複数のポリシーを&#x200B;*許可なし*&#x200B;と&#x200B;*ロック*&#x200B;に設定します。 Elissaは、選択した組織として&#x200B;*Acme Division*&#x200B;を選択すると、後でポリシー値のロックを解除して変更できますが、組織&#x200B;*Acme Division*&#x200B;によってポリシーがロックされているため、RobertとSarahは、グローバル管理者である組織のポリシーを解除できません。

## ポリシーの詳細

### 組織管理

| ポリシー名 | 説明 |
| --- | --- |
| **子組織の作成** | グローバル管理者が子組織を作成できるようにします。 *off*&#x200B;の場合、子組織は作成できません。 |
| **組織の名前を変更** | 許可されている場合、グローバル管理者またはシステム管理者は組織の名前を変更できます。 また、組織の国/地域の変更も制御します。 親組織の名前を変更したり、組織または組織の祖先を変更したりする場合は、このポリシー設定とは別に組織のパス名を変更することもできます。 |
| **組織の削除** | グローバル管理者が子組織を削除できるようにします。 これは、ユーザーアセットを削除するリスクがあるため、エンタープライズストレージを持つ組織が有効になっている場合に、より重要になります。 |

### 管理者管理

| ポリシー名 | 説明 |
| --- | --- |
| **管理者の追加または削除** | グローバル管理者が新しい管理者を組織に追加できるようにします。 *off*&#x200B;の場合、新しい管理者を追加できません。 |
| **子組織の作成時にシステム管理者を親から継承** | グローバル管理者が新しい子組織を作成すると、親のシステム管理者が新しい組織のシステム管理者になります。 このポリシーは、デフォルトで&#x200B;*off*&#x200B;です。 |
| **管理者の管理** | グローバル管理者が管理者権限を変更または削除/編集できるようにします。 |

### ユーザー管理

| ポリシー名 | 説明 |
| --- | --- |
| **親組織が管理するディレクトリからユーザーを継承** | このポリシーは、*on*&#x200B;に切り替えてアクティブにしてから、新しい子組織を作成する必要があります。 子組織が作成されると、親組織内のユーザーは子組織内のユーザーとして使用できるようになります。 つまり、このポリシーは、新しい子がGAC内で作成されたときに、親と子の間の信頼関係を自動的に設定します。 既存の組織については、GACに追加される前の信頼関係は、GACに取り込まれたままになります。 信頼関係がない場合は、通常の信頼要求プロセスに従う必要があります。 このポリシーを成功させるには、新しい組織を作成するグローバル管理者も、クレームされたドメインを持つ親組織のシステム管理者である必要があります。 そうでない場合、ドメイン信頼関係は新しく作成された組織に継承されません。 |
| **Adobe ID ユーザーを追加** | 設定されている場合、組織はAdmin Console、User Management API （UMAPI）、または同期メカニズムを介してAdobe ID タイプのユーザーを追加できません。 |
| **ユーザーグループの管理** | 許可されている場合、グローバル、システム、およびユーザーグループの管理者は、ユーザーグループを作成、編集、および削除できます。 |

### ディレクトリとドメインの適用

| ポリシー名 | 説明 |
| --- | --- |
| **クレーム ドメイン** | 設定した場合、システム管理者はAdmin Consoleでドメインを要求できます。 |
| **ID設定の変更** | 設定した場合、システム管理者はAdmin ConsoleのユーザーID設定の設定を変更できます。 |

### 製品配分

| ポリシー名 | 説明 |
| --- | --- |
| **製品の管理** | グローバル管理者が製品を追加または削除し、製品リソースの付与を変更できるようにします。 |

### アセットの共有

| ポリシー名 | 説明 |
| --- | --- |
| **システム管理者またはストレージ管理者は、アセット共有設定を変更できます** | 許可されている場合、ストレージ管理者とシステム管理者は、セキュリティ連絡先、パスワードポリシー、ストレージポリシーなどのアセット共有設定を変更できます。 |
| **組織の作成時に親から共有ポリシーを継承** | 許可されている場合、アセット共有設定は、子組織の作成時に親から継承されます。 アセット共有の設定には、セキュリティの連絡先、パスワードポリシー、ストレージポリシーが含まれます。 これは、作成時に新しく作成された組織にのみ適用されます。 これは親に設定され、その親の下にある子組織の作成に影響します。 |
