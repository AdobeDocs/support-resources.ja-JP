# 組織の構造と製品の割り当てのエクスポートまたはインポート

**適用先：** Enterprise

Global Admin Consoleの書き出しと読み込み機能を使用して、グローバル管理者が組織と製品の管理を合理化する方法について説明します。

**[!UICONTROL 2&rbrace;Global Admin Console]** の「組織」タブにアクセスして、組織構造をエクスポートまたはインポートします [&#x200B; &#x200B;](https://helpx.adobe.com/jp/enterprise/global-admin-console/adopt-global-administration.html)配分データについては、「**[!UICONTROL 製品配分]**」タブに移動します。 **[!UICONTROL その他のオプション]**&#x200B;**⋮** アイコンを使用して、書き出しまたは読み込みを選択します。 [Global Admin Consoleにログインします &#x200B;](https://global-admin-console.adobe.com)。

## 組織構造のエクスポート

[&#x200B; グローバル管理者 &#x200B;](https://helpx.adobe.com/jp/enterprise/global-admin-console/manage-administrators.html) は、組織階層を書き出すことができます。 組織階層全体またはそのサブセットの JSON、CSV、XLSX 表現をダウンロードできます。 その後、このデータを分析や変更に使用できます。

選択した書き出し形式は、書き出されたデータの構造に影響します。

- CSV 形式 – 一度に 1 種類のデータのみを書き出すことができます。 製品プロファイルを CSV 形式で書き出す場合、プロファイルとリソースが 1 つのテーブルに結合されます。 製品プロファイルには複数のエントリがあり、リソースごとに 1 つずつ存在します。
- XLSX 形式 – 結果として各組織の詳細が別々のシートに表示されます。 レコードは、参照 ID によって異なるオブジェクトタイプ間で接続されます。 場合によっては、特定のオブジェクト（特定のリソースに関連付けられた値のセットがある場合のリソースオブジェクトなど）に対して複数の行が存在する場合があります。
- JSON 形式 – 最も柔軟性が高い。 書き出されたオブジェクト間の構造の関係を利用できます（例えば、組織内のプロダクトが組織要素に直接表示される）。 同じフィールドが 3 つの形式すべてで書き出されますが、一部の値は JSON 形式で冗長になります。

### 書き出し手順

1. [Global Admin Console](https://global-admin-console.adobe.com/) にサインインします。 「**[!UICONTROL 組織]**」タブで、組織ピッカーを使用して書き出す組織階層を選択します。 階層内のすべての組織のデータが書き出されます。
2. 「**[!UICONTROL その他のオプション]**」⋮イコンを選択し、「**[!UICONTROL 書き出し]**」を選択します。

   ![&#x200B; 組織構造をエクスポート &#x200B;](./assets/export-org-structure.png)

3. **[!UICONTROL 書き出し]** ダイアログボックスで、書き出すもの、およびデータを書き出す形式を選択します。

   ![Admin Consoleの書き出しダイアログボックス &#x200B;](./assets/export-12.png)

4. **[!UICONTROL 書き出し]** を選択します。 エクスポートファイルの生成には、数分かかる場合があります。 完了したら、**[!UICONTROL Global Admin Console]**/**[!UICONTROL インサイト]**/**[!UICONTROL レポートの書き出し]** に移動してレポートをダウンロードします。

&#x200B;> [!NOTE]
>
> JSON ファイルは zip 形式で書き出されます。 zip ユーティリティまたはオペレーティングシステムの zip 機能を使用して開くことができます。

ファイルをダウンロードした後、データを操作してインポートすることができます。 読み込まれた更新は、手動でデータを編集したかのようにGlobal Admin Consoleに表示されます。

## 組織構造の読み込み

[&#x200B; グローバル管理者 &#x200B;](https://helpx.adobe.com/jp/enterprise/global-admin-console/manage-administrators.html) は、変更された可能性があるデータを読み込むことができます。 アップロードされると、新しいデータが現在のデータと比較され、変更が組織階層に適用されます。 すべてのインポート操作は、組織階層の更新されたコピーに対して実行されます。 保留中の変更がある場合、読み込みの変更は階層内の保留中の変更の上に追加されます。

### 読み込み手順

1. [Global Admin Console](https://global-admin-console.adobe.com) にサインインします。 「**[!UICONTROL 組織]**」タブで、組織ピッカーを使用して、インポートを実行する組織階層を選択します。
2. **[!UICONTROL その他のオプション]** **⋮** アイコンを選択し、「**[!UICONTROL 読み込み]**」を選択します。 インポートファイルのサイズと複雑さによっては、処理に数秒から数分かかる場合があります。
3. **[!UICONTROL ファイルを選択]** を選択し、アップロードする JSON、CSV、XLSX ファイルを選択します。 CSV の場合、一度に 1 つの組織の詳細のみを読み込めるので、製品の読み込みはサポートされません。 読み込まれた変更は、データを手動で編集した場合と同じように表示されます。
4. **[!UICONTROL 閉じる]** を選択します。
5. 「**[!UICONTROL 保留中の変更を確認]**」を選択します。 次に、「変更を送信 **[!UICONTROL を選択して]** 実行 [&#x200B; し &#x200B;](https://helpx.adobe.com/jp/enterprise/global-admin-console/execute-jobs.html) す。 保留中のアクションは、変更を実行する前に、Global Admin Consoleで手動で編集した場合と同じ方法で表示されます。

## スキーマのエクスポートとインポート

CSV ファイルを使用してデータを読み込む際、フィールドは任意の順序で表示できますが、常にヘッダー行と一致する必要があります。

データのインポート中に、各要素に対して操作を指定する必要があります。 操作は次のいずれかになります。

- 更新：編集を示します。
- 作成：新しいオブジェクト（組織、ユーザーグループ、管理者など）を作成することを示します。
- 削除：オブジェクト（組織、ユーザーグループ、管理者など）が削除されたことを示します。

操作フィールドがないか空白の入力レコードは無視されます。

**読み込みの要件：**

- 更新または削除の場合、orgId は階層内の既存の組織を参照する必要があります。
- 新しい組織を作成する場合は、「orgId」フィールドを空白のままにするか、作成可能な一意の ID （new-1 や new-2 など）に設定できます。 これにより、作成する組織を参照するために使用できる ID が提供されます。
- 国コードは有効である必要があります。
- *更新* および *削除* 操作の orgId は、既に組織階層に存在している必要があります。
- *削除* としてマークされた orgId を、*更新* または *作成* 操作を持つ組織の parentOrgId として選択しないでください。
- 同じレベルで、同じ親の子組織を同じ orgName にすることはできません。
- 組織を作成したり、組織名を更新したりする場合、その組織名が、同じ親の既存の子の名前と一致しないようにする必要があります。

### 製品プロファイル

製品プロファイルのエクスポートとインポートは、製品プロファイルの詳細と、製品プロファイルに関連付けられた一連のリソースの 2 つの部分で構成されます。 これらのリソースは、通常、サービスを有効または無効にするためだけに、設定可能なサービスを識別します。

- リソースオブジェクトは、JSON 形式の製品プロファイル内にネストされます。
- 製品プロファイルで CSV または XLSX を使用する場合、プロファイルとリソースは 1 つのテーブルに結合されます。 製品プロファイルには複数のエントリがあり、リソースごとに 1 つずつ存在します。
- リソースの「選択済み」フィールドは、サービスが有効かどうかを制御します。
- 製品プロファイルをインポートする場合は、製品プロファイル自体と、更新または作成するリソースオブジェクトに対して作成または更新の操作が必要です。

**読み込みの要件：**

- productProfileId、licenseId および orgId には、有効な値が必要です。
- 製品プロファイルを作成する場合、productProfileName は有効な名前である必要があり、同じ組織内の別の製品プロファイル名またはユーザーグループ名と重複してはなりません。
- クォータ フィールドには、ユニット タイプに有効な値が必要です。 resourceType=QUOTA の場合は数値、それ以外の場合は空白です。
- 通知フィールドは、true または false にする必要があります。
- CSV および XLSX の読み込みの場合、productProfileId を検証します。すべてのエントリは、同じ orgId、licenseId および productProfileName を持つ必要があります。
- 入力ファイルと組織で重複している productProfileName を検証します。
- 更新および削除するプロファイルは、組織内に存在する必要があります。
- 更新および削除（非アクティブ化）するリソースが、プロファイル内に存在している必要があります。
- プロファイルを作成する場合、次の点を確認します。
   - orgId は、新しい組織または既存の組織にする必要があります。
   - licenseId は、新製品または既存の製品である必要があります。
   - プロファイルのリソースを検証します。

[&#x200B; グローバル管理者 &#x200B;](https://helpx.adobe.com/jp/enterprise/global-admin-console/manage-administrators.html) は、製品割り当てデータを JSON ファイルまたは CSV ファイルとして書き出すことができます。 その後、このデータを操作し、アップロードして戻り、変更を読み込むことができます。 変更された可能性があるデータがアップロードされると、新しいデータが現在のデータと比較され、変更が製品割り当てデータに適用されます。 その後、保留中の変更を確認して送信し、有効にすることができます。

## 製品配分のエクスポートおよびインポート形式

書き出し形式と読み込み形式は同じです。 CSV 形式で読み込む場合、フィールドは任意の順序で表示できますが、ヘッダー行と一致する必要があります。 JSON 形式で読み込む場合、フィールドは任意の順序で表示できます。

製品配分データをインポートする際は、操作を指定する必要があります。 次のいずれかの操作を行うことができます。

- 更新：編集（grantedQuantity、allowOverAllocation 値への変更）を示します。
- 作成：指定した組織に製品リソースを追加することを示します。
- 削除：製品の削除を示します。

操作を指定しない場合、CSV の行または JSON のオブジェクトにデータを読み込んでも、変更は行われません。

書き出されたファイルには、製品リソースごとに 1 つの行またはレコードがあります。 一部の製品には、複数のリソースがあります。

プロダクトに複数のリソースがある場合、更新操作は独立したリソースに適用でき、削除操作は組織のすべてのリソースを含むプロダクトを削除します。また、作成操作は、各リソースの適切な数量を指定できるように、インポートファイル内の各リソースにレコードを必要とします。 allowOverAllocation フィールドは製品全体であり、このフィールドの更新がどのリソースにあるかは関係ありません。

**データの検証**

- 操作フィールドには、有効な操作が必要です。
- 製品のインポートデータには、必須フィールドのプロパティと値が必要です。

- 「grantedQuantity」フィールド：
   - まだ *unlimited* でない場合は、*unlimited* に変更できません。
   - 負でない整数または文字列値である必要があります *unlimited.*

**権限/アクセス可能な検証**

- インポートデータに関連付けられた組織が存在する必要があります。 更新する場合は、インポートデータに関連付けられた製品とリソースが実際に存在することを確認します。

**製品検証の追加**

- SourceLicenseId が存在する必要があります。
- 新しい製品に関連付けられた組織が存在する必要があります。
- 作成する製品は存在しないでください（同じ licenseId を持つ製品）。
- 作成するプロダクトに関連付けられたリソースには、そのプロダクトに一致する productId が対応していなければなりません。
