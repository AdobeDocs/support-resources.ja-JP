# 監査ログのダウンロードとレポートのエクスポート

エンタープライズに適用します。

グローバル管理者がGlobal Admin Consoleを使用して監査ログとレポートを表示、フィルタリング、書き出す方法について説明します。

開始するには、[Global Admin Console](https://global-admin-console.adobe.com/) にサインインしてください。 次に、「**[!UICONTROL インサイト]**」タブに移動し、「**[!UICONTROL 監査ログ]**」を選択して、組織全体で行われたすべての変更を追跡します。

## 監査ログの表示とダウンロード

グローバル管理者は、Global Admin Consoleで行われた変更を完全に可視化できます。 過去 90 日間に実行されたアクションを、すべての組織をまたいで監査ログで検索できます。アクションには、実行された日時や実行者も含まれます。
- 監査ログは、不適切なシステムアクセスを防ぎ、組織内の不審な動作を監査することで、継続的なコンプライアンスの確保に役立ちます。
- Global Admin Consoleで使用できるログには、グローバル管理者がアクセスできるイベントのみが含まれます。 ユーザー割り当てやユーザーイベントは含まれません。 各コンソールが提供する様々な機能については、[&#x200B; 詳細情報 &#x200B;](https://helpx.adobe.com/jp/enterprise/using/admin-console.html) を参照してください。
- ログは、階層内のすべての組織のイベントをカバーしているので、すべての組織の監査ログを一度に検索できます。

>[!NOTE]
>
> [Adobe Admin Console](https://adminconsole.adobe.com) 組織のシステム管理者は、[&#x200B; 監査ログ &#x200B;](https://helpx.adobe.com/enterprise/using/audit-logs.html) を使用してユーザーの割り当てとユーザーイベントを確認できます。 選択した組織の子組織のシステム管理者が実行したアクションも監査ログに含まれます。 システム管理者がAdmin Consoleで行われた [&#x200B; 変更をトラッキング &#x200B;](https://helpx.adobe.com/enterprise/using/audit-logs.html) する方法について説明します。

組織の監査ログを表示またはダウンロードするには：

1. グローバル管理者として、[Global Admin Console](https://global-admin-console.adobe.com/insights) にログインします。
1. **[!UICONTROL インサイト]**/**[!UICONTROL 監査ログ]** を選択します。
監査ログには、フィルターされたイベントに関する次の情報が表示されます。

   | フィールド | 説明 |
   |------ |-------------|
   | 日付 | イベントの日時。ローカルタイムゾーンで表示されます。 |
   | イベント名 | 実行されたアクションの説明。 |
   | イベントの詳細 | 追加のイベントの詳細（使用可能な場合）。 |
   | オブジェクト名 | 必要に応じて、イベントに関係する製品、製品プロファイルまたはユーザーグループの名前。 |
   | 影響を受けるユーザー | 影響を受けるユーザーのメールアドレス（該当する場合）。 |
   | 管理者 | アクションを実行した管理者の電子メールアドレス。 *システム* は、アクションがAdobe バックエンドシステムによって実行された場合に表示されます。 |
   | IP アドレス | アクションが実行されたマシンの IP アドレス。 通常は物理的な場所を反映しますが、プロキシサーバーまたは VPN アドレスの場合もあります。 |
   | 組織 | イベントの影響を受けた組織の名前。 |


1. 次のオプションを使用して、監査ログをフィルタリングできます。

   - 影響を受けるユーザーまたは管理者で検索します。
   - 1 つ以上の組織を選択します。
   - 日付範囲を定義します。
   - イベント名でフィルタリングします。
   - フィルターを組み合わせて、特定の組織の過去 7 日間のイベントを表示するなど、結果を絞り込むことができます。

   ![audit-logs](assets/audit-logs.png)

   *イベント名、影響を受ける組織または日付範囲に基づいて監査ログをフィルタリングします*。

   ![select-organization](assets/select-organizations.png)

   *監査ログをフィルタリングする組織を選択します。*

1. 監査ログを書き出すには、「**[!UICONTROL CSV を書き出し]**」を選択して、フィルタリングされた結果を書き出します。 結果は CSV 形式でダウンロードされます。

エクスポートに含まれるフィールドについて詳しくは、[&#x200B; ログスキーマ &#x200B;](#log-schema) を参照してください。

>[!NOTE]
>
>書き出された監査ログレポートは、生成後 90 日間保持されます。


## 監査ログレポートについて {#log-schema}

書き出された監査ログレポートには、各組織に関する次のフィールドが含まれています。

| フィールド | 説明 |
|------|------------|
| ID | イベント ID |
| eventTime | イベントの日時（ローカルタイムゾーン） |
| eventType | イベント名 |
| eventSubType | 追加のイベントの詳細（利用可能な場合） |
| actorEmail | イベントを実行した管理者の電子メールアドレス。 *システム* は、イベントがAdobe バックエンドシステムによって実行された場合に表示されます。 |
| targetUserEmail | 影響を受けるユーザーのメール（該当する場合） |
| targetGroupName | 影響を受けるユーザーグループ（該当する場合） |
| targetProductName | 該当する製品（該当する場合） |
| targetProfileName | 影響を受ける製品プロファイル（該当する場合） |
| ipAddress | アクションが実行されたマシンの IP アドレス。 通常は物理的な場所を反映しますが、プロキシサーバーまたは VPN アドレスの場合もあります。 |
| organizationName | 影響を受ける組織の名前 |

## 書き出しレポートのダウンロード

任意のグローバル管理者が [Global Admin Console](https://global-admin-console.adobe.com/insights) から組織データを書き出すと、レポートは処理され、**[!UICONTROL レポートの書き出し]** の **[!UICONTROL インサイト]** タブで使用できるようになります。

グローバル管理者が生成したすべてのレポートが 1 か所で利用できます。 レポートの書き出し機能は、次のシナリオで役立ちます。

- 多数の組織を持つ大きな階層がある場合、エクスポートファイルが処理されるのを待つ必要がなくなりました。 管理者が生成したレポートを使用できます。
- 後で比較するために、これらのレポートを保持または保存する必要はなくなりました。 レポートは 90 日間保持され、いつでもダウンロードしてレポートを比較できます。


エクスポートレポートをダウンロードするには：

1. [Global Admin Console](https://global-admin-console.adobe.com/insights) にログインし、**[!UICONTROL インサイト]**/**[!UICONTROL レポートの書き出し]** に移動します。

   過去 90 日間に生成されたレポートが表示されます。 90 日が完了したら、レポートを再度生成できます。 [&#x200B; 組織構造 &#x200B;](https://helpx.adobe.com/enterprise/global-admin-console/export-and-import-data.html#export-and-import-organization-structure) のレポートを生成する方法を説明します。


   | フィールド | 説明 |
   |------|------------|
   | レポート | レポートが生成された日時（ローカルタイムゾーン） |
   | 形式 | ファイル形式（CSV、JSON、XLSX） |
   | サイズ | ファイルサイズ |
   | 作成者 | レポートを生成した管理者のメールアドレス |
   | アクション | レポートをダウンロードするためのリンク |

1. レポートをダウンロードするには、「**[!UICONTROL ダウンロード]**」を選択します。

   生成したレポートがリストに表示されない場合は、「**[!UICONTROL 更新]**」を選択します。

![export-reports](assets/export-reports.png)

*過去 90 日間に生成されたレポートをダウンロードします。*
