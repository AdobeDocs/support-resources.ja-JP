# Adobe Admin Consoleの概要

エンタープライズ版とグループ版に適用。

Adobe Admin Console は、組織全体のアドビ製品エンタイトルメントを一元的に管理する場所です。 ライセンス、ユーザー、支払いの管理に使用できます。 [Admin Consoleにログインするには、こちらに移動します](https://adminconsole.adobe.com)。 詳しくは、次の[video](https://helpx.adobe.com/jp/enterprise/using/admin-console.html)を参照してください。

Admin Consoleの各タブでは、様々なタスクを実行できます。 詳細については、以下のタイトルを選択してください。

- [!UICONTROL 概要] {#overview}：購入したライセンスの概要と、組織をセットアップするためのクイックアクションを表示します。

- [!UICONTROL 製品] {#products}: ユーザーとグループにライセンスを割り当てます。 エンタープライズ版のお客様は、製品プロファイルを管理できます。

- [!UICONTROL Users] {#users}: ユーザーアカウントを作成、更新、削除し、エンドユーザーにAdobe製品とサービスの使用権限を付与します。

- [!UICONTROL &#x200B; パッケージ &#x200B;] {#packages}：事前設定済みのパッケージをダウンロードするか、デプロイするデスクトップアプリ用に作成します。

- [!UICONTROL &#x200B; アカウント &#x200B;] {#account}：組織の契約と契約書をAdobeで管理します。

- [!UICONTROL &#x200B; ストレージ &#x200B;] {#storage}：個々のユーザーフォルダーと共有フォルダーを管理し、ユーザーが使用したストレージの割り当てを表示します。

- [!UICONTROL &#x200B; インサイト &#x200B;] {#insights}: ライセンス割り当てレポートを表示、作成、ダウンロードし、Admin Consoleで行われた変更を追跡します。

- [!UICONTROL 設定] {#settings}: ドメインの要求、共有機能へのアクセスの制限、エンドユーザーのメモの追加、パスワード保護レベルの設定を行います。

Admin Consoleにログインできない場合は、[Adobe アカウントのログインに関する問題を解決](https://helpx.adobe.com/manage-account/kb/account-password-sign-help.html)を参照してください。

## [!UICONTROL 概要] {#overview}

「**[!UICONTROL 概要]**」タブには、製品ライセンスに関する豊富な情報が効率的に表示されます。 プラン内のライセンスのステータス（使用可能なライセンスの合計数のうち、割り当てられたライセンス数）が表示されます。 また、ユーザーと管理者を追加するためのクイックリンクも利用できます。

## 組織の選択

管理者は複数の組織に属することができます。 会社が複数の子会社を持ち、その子会社が別々の組織として存在する場合、または各子会社に個別のライセンス契約がある場合、同じ管理者をすべてに割り当てることができます。

複数の組織の管理者である場合は、組織セレクターを使用して組織を切り替えることができます。 選択した組織には、組織名の横に緑色のチェックマークが表示されます。

![管理コンソール組織を選択](assets/admin-console-select-your-org.png "組織セレクター")

組織がGlobal Admin Consoleの一部である場合、組織名の横に階層アイコンが表示されます。 組織のパスも表示され、階層内での組織の配置を決定できます。 例えば、スクリーンショットでは、管理者は組織Bのメンバーであり、この組織のGlobal Admin Console パスはA > Bです。Bは組織Aの子です。

![管理コンソール子の組織を選択](assets/admin-console-select-your-org-hierarchy.png "Global Admin Consoleの階層内の組織プレースメント ")

多くのAdmin Consoleを持つ複雑な組織構造がある場合、またはメインのAdmin Consoleを複数のコンソールに分割する場合は、[Global Admin Consoleを導入できます](https://helpx.adobe.com/enterprise/global-admin-console/adopt-global-administration.html)。 たとえば、多国籍企業、教育コンソーシアム、大規模な学区、大規模な政府機関などです。 Global Admin Consoleでは、既存のAdmin Consoleを組織図などの階層構造にネストして、分散したエンタープライズ全体に透明性を提供します。

## [!UICONTROL 製品] {#products}

このタブを表示できるユーザー：システム管理者、製品管理者、製品プロファイル管理者

エンタープライズ {#enterprise}


**[!UICONTROL Admin Console]**&#x200B;の[製品](https://adminconsole.adobe.com) ページには、製品と製品プロファイルを管理するためのオプションが用意されています。 製品プロファイルを使用すると、プランで利用可能なAdobe アプリケーションとサービスのすべてまたはサブセットを有効にし、特定の製品またはプランに関連する設定をカスタマイズできます。 次に、製品管理者という名前の管理者を製品プロファイルに割り当てることができます。 これらの管理者は、管理する製品プロファイルにエンドユーザーを追加します。

詳しくは、次を参照してください。

- [製品の管理](https://helpx.adobe.com/jp/enterprise/using/manage-products.html)
- [製品プロファイルの管理](https://helpx.adobe.com/jp/enterprise/using/manage-product-profiles.html)

チーム {#teams}

**[!UICONTROL Admin Console]**&#x200B;の[製品](https://adminconsole.adobe.com) ページでは、製品ライセンスをユーザーに割り当てることができます。 製品ライセンスをユーザーまたはグループに割り当てるには、**[!UICONTROL 製品]** ページで目的の製品を選択し、**[!UICONTROL ユーザーを追加]**&#x200B;をクリックします。

ユーザーの名前または電子メールアドレスを入力します。 有効なメールアドレスを指定し、画面に表示される情報を入力することで、既存のユーザーを検索したり、ユーザーを追加したりできます。 「**[!UICONTROL 保存]**」をクリックします。アプリケーションへのアクセスを確認するメールがユーザーまたはグループに送信されます。

詳しくは、次を参照してください。

- [&#x200B; ライセンスの割り当てまたは割り当て解除](https://helpx.adobe.com/enterprise/using/assign-licenses-to-teams-users.html)
- [製品またはライセンスの追加または削除](https://helpx.adobe.com/enterprise/using/add-products-and-licenses.html)

## [!UICONTROL ユーザー] {#users}

**[!UICONTROL Admin Console]**&#x200B;の[Users](https://adminconsole.adobe.com) ページでは、ユーザーアカウントを作成、検索、更新、削除できます。 これらのユーザーアカウントは、組織内のエンドユーザーにAdobeの製品とサービスを提供する権利を与えます。 また、一括編集ワークフローを使用して、ユーザーを追加したり、ユーザーの詳細やライセンスの割り当てを変更したりすることもできます。

詳しくは、次を参照してください。

- [&#x200B; ユーザーの管理](https://helpx.adobe.com/jp/enterprise/using/users.html)
- [&#x200B; ユーザーグループの管理](https://helpx.adobe.com/jp/enterprise/using/user-groups.html)

## [!UICONTROL アカウント] {#account}

このタブを表示できるユーザー：システム管理者と契約管理者。

システム管理者と契約管理者は、組織のAdobe コントラクトをAdmin Consoleの「**[!UICONTROL アカウント]**」タブから管理できます。

Enterprise、VIP、VIP Marketplace、Teamsなどのプランに応じて、次のことができます。

- 契約ID、ステータス、契約応当日/終了日、アプリやライセンスなど、主要な契約の詳細を表示します。
- 簡単に識別できるように、契約書の表示名を変更します。
- 契約管理者を追加または削除します。
- 支払いの詳細、請求書、更新を管理します。
- Adobe アカウントマネージャーの連絡先の詳細を表示します。

詳細：[&#x200B; アカウントの管理](https://helpx.adobe.com/enterprise/using/accounts.html)。

## [!UICONTROL インサイト] {#insights}

このタブを表示できるユーザー：システム管理者。

### [!UICONTROL 監査ログ &#x200B;] {#audit-log}

**[!UICONTROL 監査ログ]**&#x200B;は、引き続きコンプライアンスを確保し、不適切なシステムアクセスを保護し、組織内の疑わしい動作を監査するのに役立ちます。

システム管理者は、[Admin Console](https://adminconsole.adobe.com/)で行われた変更を完全に把握できます。 監査ログは、アクションのタイプ、アクションが発生したタイミング、アクションを実行したユーザーに基づいて検索できます。

次に、これらのレポートを表示してダウンロードし、さらに分析します。 詳細：[監査ログを使用して、ユーザーの割り当てとイベントを追跡する](https://helpx.adobe.com/enterprise/using/audit-logs.html)。

### [!UICONTROL 割り当てレポート &#x200B;] {#assignment-reports}

ライセンス割り当てレポートを使用すると、組織のライセンス割り当てデータを追跡し、ユーザーのライセンスのデプロイメントを計画できます。 ライセンス割り当てデータは、エンタープライズタームライセンス契約に基づいて購入されたCreative CloudおよびDocument Cloud製品のユーザー指定ライセンスのみをサポートします。

詳細：[&#x200B; エンタープライズ製品のライセンス割り当てレポート &#x200B;](https://helpx.adobe.com/enterprise/using/assignment-reports.html)。

## [!UICONTROL &#x200B; ストレージ &#x200B;] {#storage}

このタブを表示できるユーザー：システム管理者とストレージ管理者（[&#x200B; プールされたストレージモデル &#x200B;](https://helpx.adobe.com/enterprise/using/manage-adobe-storage.html)に移行された顧客のみ）。

**[!UICONTROL Creative Cloud]**&#x200B;の[&#x200B; ストレージ ページ &#x200B;](https://adminconsole.adobe.com)は、Admin Console アプリケーション全体のストレージを表示します。 ストレージの割り当ては、組織が購入するストレージの量まで、エンドユーザーに対して柔軟です。

また、個々のユーザーが使用する割り当て量と、すべてのユーザーが使用する全体的な割り当て量を表示することもできます。

詳細：[Adobe ストレージの管理](https://helpx.adobe.com/enterprise/using/manage-adobe-storage.html)。

## [!UICONTROL &#x200B; パッケージ &#x200B;] {#packages}

このタブを表示できるユーザー：システム管理者とデプロイメント管理者。

**[!UICONTROL Admin Console]**&#x200B;の[&#x200B; パッケージ &#x200B;](https://adminconsole.adobe.com) ページには、次の機能が用意されています。 組織内のエンドユーザーにデスクトップアプリケーションをデプロイする場合に使用します。

- [Adobe テンプレート &#x200B;](https://helpx.adobe.com/enterprise/using/package-templates.html)を使用して、事前設定済みのパッケージをダウンロードします。
- カスタマイズされた[&#x200B; ユーザー指定ライセンス &#x200B;](https://helpx.adobe.com/enterprise/using/create-nul-packages.html)または[共有デバイス &#x200B;](https://helpx.adobe.com/enterprise/using/create-sdl-packages.html) ライセンス （教育機関向け）のパッケージャーを、エンドユーザーが持つ設定とアプリケーションを使用して作成します。
- メール通知を有効にすると、新しい製品バージョンが利用可能になったときに通知を受け取ることができます。
- 自分または組織内の他の管理者が作成した以前のパッケージを表示します。 また、特定のパッケージの詳細を表示し、パッケージ内のアプリに関する利用可能な更新を追跡します。
- [Remote Update Manager](https://helpx.adobe.com/enterprise/using/using-remote-update-manager.html)や[Adobe Update Server Setup Tool](https://helpx.adobe.com/enterprise/using/update-server-setup-tool.html)などのIT ツールをダウンロードします。
- Adobe Extension Manager コマンドラインツールをダウンロードして、ZXP ファイルコンテナフォーマットから[拡張機能とプラグイン &#x200B;](https://helpx.adobe.com/enterprise/using/manage-extensions.html)をインストールします。

詳しくは、「[Admin Consoleを介したアプリのパッケージ化](https://helpx.adobe.com/enterprise/using/package-apps-admin-console.html)」を参照してください。

## [!UICONTROL 設定] {#settings}

このタブを表示できるユーザー：システム管理者とストレージ管理者。

ストレージ管理者がアクセスできるのは、[&#x200B; アセット設定](https://helpx.adobe.com/enterprise/using/asset-settings.html)および[&#x200B; コンテンツログ &#x200B;](https://helpx.adobe.com/enterprise/using/content-logs.html)のみです。 システム管理者は、プランに応じて設定を表示または変更できます。

>[!NOTE]
>
> Adobeでは、トップレベルの管理者が現在のAdmin Console設定をAdobeの推奨されるセキュアデフォルトと比較するためのネイティブ機能を提供していません。 管理者は、Adobeの推奨設定ガイダンスを参照し、組織のID プロバイダー、エンドポイント管理ツール、内部監査プロセスを使用してコンプライアンスを検証できます。

## プライバシーとセキュリティの連絡先 {#privacy-and-security-contacts}

当社のソフトウェアソリューションに関するセキュリティインシデントが発生した場合、適切なコンプライアンス担当者に通知が送信されます。 迅速な通知を実現するには、システム管理者として、セキュリティ、データ保護、コンプライアンスに関する担当役員を特定する必要があります。 詳しくは、[&#x200B; プライバシーとセキュリティに関する連絡先](https://helpx.adobe.com/enterprise/using/security-contacts.html)を参照してください。

## [!UICONTROL &#x200B; コンソール設定] {#console-settings}

[[!UICONTROL &#x200B; コンソール設定]](https://helpx.adobe.com/enterprise/using/console-settings.html)を使用すると、問題が発生した場合やサポートが必要な場合にサポートを受ける方法について、エンドユーザーにカスタムノートを追加できます。

サブスクリプションの変更やクレジットカードの有効期限など、アカウントのステータスに関するメールを受信するには、組織のデフォルトのメール言語を選択します。 Adobeから直接購入したグループ版メンバーシップがある場合は、**[!UICONTROL コンソール設定]**&#x200B;からグループ名を変更できます。

## [!UICONTROL &#x200B; コンテンツログ &#x200B;] {#content-logs}

管理者は、エンドユーザーがフォルダー、ファイル、ライブラリなどの企業アセットをどのように使用しているかに関する詳細なレポートをダウンロードできます。 これらのレポートは[[!UICONTROL &#x200B; コンテンツログ &#x200B;]](https://helpx.adobe.com/enterprise/using/content-logs.html)と呼ばれます。

## ドメインの適用 {#domain-enforcement}

システム管理者は、組織が所有するドメインを制限して、ユーザーが個人のAdobe ID アカウントを作成および使用できないようにすることができます。 これにより、個人データの使用を制限し、セキュリティを強化し、組織ユーザー間でのみアセットを共有できます。

詳細：[制限付き認証のドメインの適用](https://helpx.adobe.com/enterprise/using/restricting-domains.html)。

## ID {#identity}

[ID タイプ &#x200B;](https://helpx.adobe.com/jp/enterprise/using/identity.html)を使用すると、組織はユーザーのアカウントとデータに対して異なるレベルの制御を行うことができます。 組織でのアセットの保存および共有方法に影響します。

## [!UICONTROL &#x200B; アセット設定] {#asset-settings}

[&#x200B; アセット設定](https://helpx.adobe.com/enterprise/using/asset-settings.html)により、組織は、従業員が組織外でアセットを共有する方法を制御できます。 アセット設定は、他の組織ポリシー適用システム（Adobeでは提供されていません）と共に使用され、適切な外部の個人や組織とのみアセットを共有するように設定されます。

## 認証設定 {#authentication-settings}

[認証設定](https://helpx.adobe.com/enterprise/using/authentication-settings.html)では、安全性とセキュリティを確保するために、いくつかのパスワード保護レベルとポリシーがサポートされています。 パスワード保護レベルを指定して、組織のすべてのユーザーに適用できます。

## 暗号化設定 {#encryption-settings}

[暗号化設定](https://helpx.adobe.com/enterprise/using/encryption.html)は、制御とセキュリティの追加レイヤー用に専用の暗号化キーを生成します。

## プロジェクトポリシー {#project-policies}

システム管理者は、組織内のプロジェクトを作成および管理する権限を持つユーザーを制御できます。 デフォルトでは、Admin Consoleに追加されたすべてのユーザーがプロジェクトを作成および管理できます。

詳細：[&#x200B; プロジェクトポリシー](https://helpx.adobe.com/enterprise/using/projects-in-business-storage.html#project-policies)。

## サポート

Adobe カスタマーケアにお問い合わせするには、[Admin Console](https://adminconsole.adobe.com/)のサポートページに移動し、次の操作を行うことができます。

- サポートケースの管理（エンタープライズ版のみ）
- ケースの作成（エンタープライズ版のみ）
- Adobeのカスタマーサービス部門との連携
- エキスパートセッションのスケジュール
- 人気のヘルプトピックとフォーラムを参照する

サポートオプションについて詳しくは、[&#x200B; サポートとエキスパートセッション &#x200B;](https://helpx.adobe.com/enterprise/using/support-and-expert-services.html)を参照してください。
